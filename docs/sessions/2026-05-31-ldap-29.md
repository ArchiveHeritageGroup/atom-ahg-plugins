# LDAP / AD integration — wire + expose (#29)

**Date:** 2026-05-31
**Issue:** ArchiveHeritageGroup/atom-extensions-catalog#29
**Repos:** atom-ahg-plugins (ahgSettingsPlugin) + atom-framework (patches/lib)
**Status:** Built, lint-clean, settings route verified on PSIS. PSIS auth NOT switched (PSIS is not LDAP-backed). #30 SAML/SSO parked.

## Context

Base AtoM already ships a working `lib/ldapUser.class.php` (LDAP bind, TLS, search,
auto-create user, local-password fallback) + the ldapHost/Port/BaseDn/BindAttribute
settings + a login form that adapts when the user class is `ldapUser`. So #29 was ~80%
done — the gap was a runtime enable toggle, a connection test, and a safe way to switch
the user class. Decision (Johan): scope 1a = wire + expose existing (not a new plugin).

SSO in general already exists via arOidcPlugin (OIDC) + arCasPlugin (CAS); #30 (SAML 2.0)
needs a new library + a test IdP + security review → parked.

## Changes

### atom-ahg-plugins / ahgSettingsPlugin
- `handlers/ldapAction.class.php`: added `ldap_enabled` toggle (checkbox) to the LDAP
  settings form (saved to setting scope 'ldap').
- `templates/ldap.blade.php`: enable toggle + a CSP-safe **Test LDAP connection** form
  (plain POST, result via flash — no inline JS) + an **Activation** card documenting the
  factories.yml switch.
- `handlers/ldapTestAction.class.php` (new): connectivity/bind probe against saved settings;
  optional test user → authenticated bind; guards php-ldap + TLS; result via flash. Read-only.
- Route `/ahgSettings/ldapTest` + `executeLdapTest` delegate (mirrors ftpTest/fusekiTest).

### atom-framework / patches/lib
- `ahgLdapUser.class.php` (new): `extends ldapUser`, **safe-by-default** — `initialize()`
  only enforces php-ldap when the extension is present (else falls back to myUser bootstrap);
  `authenticate()` uses the LDAP path only when `ldap_enabled` is on, else local password.
  Switching the user class to ahgLdapUser is therefore non-breaking and reversible.

## Activation (per server, by an admin — NOT applied to PSIS)

1. Install php-ldap.
2. Copy the wrapper into lib/ (bin/install Step 11 should cp it — proposed; or manual copy).
3. `config/factories.yml` → `user.class: ahgLdapUser`.
4. Clear cache + restart php-fpm.
5. Admin > AHG Settings > LDAP authentication: configure host/port/baseDn/bindAttr, Test, Enable.
