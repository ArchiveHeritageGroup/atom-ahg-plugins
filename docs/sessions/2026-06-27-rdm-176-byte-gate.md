# RDM #176 — byte-level ODRL enforcement (approach B)

**Date:** 2026-06-27
**Issue:** ArchiveHeritageGroup/atom-ahg-plugins#176 (parity heratio#1347)
**Builds on:** the completed RDM port (epic #167)

## Problem
AtoM serves `digital_object` files as STATIC files via nginx from `/uploads`, so a
guessed/known raw URL bypassed the ODRL catalogue gate — restricted/embargoed RDM
datasets' bytes were still fetchable. The base AtoM IO viewer (LOCKED) serves
masters from the stored path, so there is no PHP byte-route to hook (unlike Heratio).

## Approach B (chosen by Johan): relocate + authed X-Accel download
- **Relocate restricted bytes off the public tree.** On restrict/embargo/de-identify,
  move every `digital_object` file of the dataset's IOs into a non-web-served
  protected dir; the old `/uploads` path 404s → raw-URL bypass closed. On open
  release, move them back.
- **Authed download controller** evaluates ODRL and serves protected files via
  `X-Accel-Redirect` from an `internal` nginx location.

## Files
- `database/install.sql` — new `rdm_protected_object` table (do_id UNIQUE; dataset_id; io_id; original_path; protected_path; moved_at). No base-table changes.
- `lib/Services/DatasetFileGuardService.php` — `protect()` / `release()` (rename with cross-FS copy+unlink fallback for the NAS), `protectedPathForDo()`, `accelUri()`. Protected dir + X-Accel prefix configurable (`app_rdm_protected_dir` / `app_rdm_protected_url`), no hardcoded paths.
- `lib/Services/DatasetReleaseService.php` — `apply()` now calls `guard->protect()` on restrict/embargo/de-identify and `guard->release()` on open release.
- `modules/rdm/actions` — `executeFileDownload()`: requireAuth; admin bypass; else `OdrlService::evaluateAccess('archival_description', ioId, researcherId, 'use')` → 403 if denied; protected → `X-Accel-Redirect` (or `readfile` fallback when `app_rdm_xaccel=false`); open → stream from `/uploads`.
- `config` — route `/research/datasets/:id/file/:fid`.
- `modules/rdm/templates/showSuccess.php` — per-file Download buttons (→ the gated controller) + an access-control note.
- `docs/nginx-rdm-protected.md` — the operator steps (protected dir + nginx `internal` location).

## Operator steps (Johan — nginx is outside the plugin)
1. `mkdir /mnt/nas/heratio/rdm-protected` (www-data:www-data, 750).
2. Add the `location /rdm-protected/ { internal; alias /mnt/nas/heratio/rdm-protected/; }` block to the PSIS nginx conf; `nginx -t && systemctl reload nginx`.
3. DB: the new `rdm_protected_object` table (run install.sql / extension reinstall).
4. (Optional) `app_rdm_xaccel: false` in config/app.yml to use the readfile fallback if the nginx location isn't added.

## Trade-off (accepted)
A restricted dataset's master also disappears from the base AtoM catalogue viewer
(it serves from `/uploads`, now empty for that file) — correct posture for
restricted/POPIA data; access is solely via the authed RDM controller. Open
datasets unaffected.
