## What shipped (4 subsystems)

- **#114 Target-journal directory** ("Where to Publish") — `research_target_journal` table,
  `TargetJournalService`, 6 per-action files in modules/research, `research:seed-target-journals`
  bin/atom command (DHET-accredited SA module; directory core jurisdiction-neutral). **12 DHET
  journals seeded.** Auto-assemble + best-fit suggest.
- **#115 Journal builder + manuscript workspace** — `research_journal/_issue/_article`,
  `ResearchJournalService`, new `researchjournal` module. Publication + manuscript modes;
  manuscript validates against the #114 directory when present.
- **#116 Lecture builder** — `research_lecture/_section/_resource`, `LectureService`,
  `LectureBuilderActionsTrait` (use-d by researchActions). type curriculum/talk/standalone;
  ordered sections + resources; status workflow.
- **#117 Training curriculum + LMS** — `training_course/_module/_assessment/_enrolment/
  _progress/_certificate`, `ResearchTrainingService`, new `training` module. Institution-neutral;
  enrol → progress → assessment → auto-scored completion → printable certificate. Modules can
  reuse #116 curriculum lectures (graceful if absent).

## How it was built

Multi-agent **workflow** (`research-portal-port`): 4 parallel build agents (one per subsystem,
mirroring `/usr/share/nginx/heratio/packages/ahg-research/src`) → per-subsystem adversarial
verify agents (re-read vs Heratio, checked data-model parity + AtoM conventions, fixed defects,
confirmed `php -l`). 8 agents, ~886k tokens. A real bug was caught + fixed in verify: #114
builder created a DUPLICATE row on edit because `AhgController::redirect()` does NOT throw (unlike
Symfony `sfAction::redirect`) — fall-through into create(); fixed with explicit `return;` +
module/action?id= redirect form.
